Detailed Description

An default HTTP implementation of the I|O Transporter that conforms with the JPPipelineSink protocol.

This default implementation use the HTTP protocol to transport data between an external server and your application, performing basic HTTP requests and interacting with REST-based services (GET / POST / PUT / DELETE).

Internally it uses the wonderful ASIHTTPRequest library. ASIHTTPRequest is builded on top of the Apple CFNetwork API and works in both Mac OS X and iPhone applications.

Once created your JPPipeline you should attach the transporter. 

 // Create new Pipeline.
 JPPipeline *pipeline = [JPPipeline init];
 
 // HTTP Transporter associated with the pipeline.
 pipeline.sink = [JPHTTPTransporter init];

HTTP Transporter Events Messages

JPDefaultHTTPMessage is an implementaton of the JPTransporterHTTPMessage protocol. It is an type of Event Message that encapsulate the HTTP data to be transported by the JPHTTPTransporter I|O implementation. You can use this class directly, create your own subclass of this event or create your own implementation of the JPTransporterHTTPMessage protocol.

An simply code to illustrate how to send an JPDefaultHTTPMessage downstream: 

 JPDefaultHTTPMessage *eventMessage = [JPDefaultHTTPMessage initWithData:dataToTransport withMethod:@"POST"]
 eventMessage.transportURL = [NSURL URLWithString:@"http://seqoy.org/httpgateway"];
 
 [pipeline sendDownstream:[JPPipelineMessageEvent initWithMessage:eventMessage]];

Of course this example require a big boilerplate for a simple operation. See Using Factories for more information to how configure and automate boilerplates using Factory Patterns.
